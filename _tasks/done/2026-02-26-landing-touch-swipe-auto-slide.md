## 작업 목표

랜딩 페이지 배너 캐러셀에 모바일 터치 스와이프 기능과 자동 슬라이드 기능을 추가한다.

## 상세 지시사항

1. **터치 스와이프 (모바일)**
   - `js/landing.js`에 터치 이벤트 핸들러 추가
   - `touchstart`, `touchmove`, `touchend` 이벤트 처리
   - 기존 마우스 드래그 로직과 일관된 동작 (좌우 스와이프로 배너 이동)
   - 스와이프 임계값(threshold) 설정 (50px 이상 이동 시 슬라이드 전환)

2. **자동 슬라이드**
   - 일정 간격(5초)마다 다음 슬라이드로 자동 전환
   - 사용자가 수동 조작(드래그, 스와이프, 화살표 클릭) 시 자동 타이머 리셋
   - 마지막 슬라이드에서 첫 슬라이드로 순환
   - 페이지 비활성(탭 전환 등) 시 자동 슬라이드 일시정지

## 참고 자료

- 기존 배너 캐러셀 구현: `js/landing.js`
- 랜딩 페이지 명세: `docs/02-page-specs/랜딩 페이지.md`
- 구현 현황: `docs/05-implementation/구현 현황.md` (1-2-1 배너 섹션)

## 제약 조건

- Vanilla JS만 사용 (외부 라이브러리 금지)
- 기존 마우스 드래그/화살표 기능이 깨지지 않아야 함
- 디자인 시스템(`css/variables.css`)의 CSS 변수 활용
- 구현 후 `docs/05-implementation/구현 현황.md`의 해당 체크리스트 업데이트

## 완료 보고

> 이 섹션은 작업을 수행하는 프로젝트 Cursor가 작성합니다.

**완료일:** 2026-02-26
**상태:** 완료

### 수행 내역

1. **터치 스와이프 개선**: 기존 터치 이벤트 핸들러에 50px 임계값(TOUCH_SWIPE_THRESHOLD) 적용. 마우스 드래그는 기존 10% 기준 유지, 터치는 50px 기준으로 분리하여 모바일 UX 최적화.
2. **자동 슬라이드 구현**: 5초 간격(AUTO_SLIDE_INTERVAL) 자동 전환. 마지막→첫 슬라이드 순환. 수동 조작(드래그, 스와이프, 화살표, 키보드) 시 타이머 리셋. visibilitychange API로 탭 비활성 시 자동 정지.
3. **구현 현황 문서 업데이트**: 터치 스와이프, 자동 슬라이드 체크리스트 완료 처리. 랜딩 페이지 진행률 90%→95% 반영.

### 변경된 파일

- `js/landing.js` — 터치 스와이프 임계값(50px) 적용, 자동 슬라이드 로직(startAutoSlide/stopAutoSlide/resetAutoSlide), visibilitychange 리스너, 모든 수동 조작에 resetAutoSlide 호출 추가
- `docs/05-implementation/구현 현황.md` — 1-2-1 배너 섹션의 터치 스와이프/자동 슬라이드 체크박스 완료, 요약 테이블 진행률 업데이트
