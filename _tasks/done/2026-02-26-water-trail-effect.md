# SP-002 — 랜딩 배너 물결 효과 개선: 수면 위 휘젓기 효과

> 발행: 2026-02-26
> 프로젝트: 센스포인트
> 우선순위: 높음
> 상태: 대기

---

## 작업 목표

랜딩 페이지 배너의 기존 물결 효과(동심원 ripple)를 **잔잔한 수면 위를 손가락으로 휘젓는 듯한 효과**로 개선한다. 마우스 포인터가 이동하는 경로를 따라 물이 일렁이는 트레일이 남아야 한다.

## 현재 상태

- `js/landing.js` 155~258줄에 기존 물결 효과 구현 (Water Ripple Effect 섹션)
- `Ripple` 클래스: 마우스 위치에서 동심원이 팽창하며 사라지는 방식
- Canvas 2D 기반, accent 색상(`rgba(200, 169, 110)`) 사용
- 마우스 이동 시 80ms 간격으로 ripple 생성

## 상세 지시사항

기존 Water Ripple Effect 섹션(155~258줄)을 교체한다.

### 원하는 효과

1. **수면 트레일 효과**
   - 마우스가 움직이는 경로를 따라 잔잔한 물결이 퍼져나감
   - 정지된 동심원이 아니라, 마우스 이동 방향/속도에 따라 물결이 흘러가는 느낌
   - 빠르게 움직이면 물결이 더 크고 넓게, 천천히 움직이면 잔잔하게

2. **시각적 느낌**
   - "잔잔한 물에 손가락을 대고 휘적휘적 젓는" 느낌
   - 물결이 마우스 경로 양옆으로 퍼지며 서서히 사라짐
   - 부드럽고 유기적인 움직임 (딱딱한 원이 아닌 자연스러운 곡선)

3. **기술적 구현 방향** (참고 — 더 나은 방법이 있으면 자유롭게 선택)
   - Canvas 2D 유지하되, 마우스 경로를 따라 여러 포인트를 찍고 각 포인트에서 양옆으로 파동 생성
   - 또는 WebGL 기반 수면 시뮬레이션 (heightmap + 물결 전파)
   - 또는 SVG 필터(feTurbulence + feDisplacementMap) 활용
   - 성능을 고려하여 최적의 방식 선택

4. **인터랙션**
   - 마우스 이동: 경로를 따라 물결 트레일
   - 클릭: 더 큰 파장의 물결 (기존 동작 유지)
   - 터치: 터치 이동에도 동일 효과
   - 마우스가 멈추면 물결이 서서히 잠잠해짐

## 참고 자료

- 기존 물결 구현: `js/landing.js` 155~258줄 (Water Ripple Effect 섹션)
- 캔버스 요소: `index.html`의 `#rippleCanvas`
- 디자인 토큰: `css/variables.css` (accent 색상 `--color-accent: #c8a96e`)
- 랜딩 페이지 명세: `docs/02-page-specs/랜딩 페이지.md`

## 제약 조건

- Vanilla JS만 사용 (외부 라이브러리 금지)
- 배너 캐러셀(드래그/스와이프/자동슬라이드)과 충돌하지 않아야 함
- 모바일에서 성능 문제가 없어야 함 (requestAnimationFrame 사용, 불필요한 연산 최소화)
- accent 색상 계열 유지 (`rgba(200, 169, 110, ...)` 또는 CSS 변수 참조)
- 배너 이미지/텍스트 가독성을 해치지 않을 정도의 은은한 효과

### 수행 내역

- `js/landing.js`의 Water Ripple Effect 섹션(기존 154~258줄)을 **Water Surface Trail Effect**로 전면 교체
- 기존 동심원 확장 방식(Ripple 클래스) → 마우스 경로 기반 수면 트레일 방식으로 개선
- **TrailWave**: 마우스 이동 경로에 타원형 파동 생성 (이동 방향에 수직으로 넓게 퍼짐)
  - 이동 속도에 비례하여 파동 크기·불투명도·링 수 자동 조절
  - 사인파 왜곡으로 유기적인 수면 느낌 구현
- **SplashWave**: 클릭 시 원형 파장 (기존 동작 유지, 더 자연스러운 물결)
- **Trail Glow**: 마우스 경로를 따라 부드러운 발광 트레일 (3단계 패스로 글로우 효과)
- 터치 이벤트 지원 (touchmove + touchend)
- mouseleave/touchend 시 포인터 추적 리셋 (이전 좌표 오염 방지)
