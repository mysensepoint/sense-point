# My Note

> 나만의 감각 노트. 자유롭게 생각을 기록하고, 모든 페이지에서 만든 메모를 한곳에서 관리하는 개인 허브.

> [!info] 구현 파일
> `my-note.html` · `css/my-note.css` · `js/my-note.js`

---

## 개요

My Note는 SensePoint의 **개인 기록 허브** 역할을 한다.

1. **노트 에디터** — 자유 형식(무지) 또는 가이드 질문(양식지)으로 깊은 생각을 기록
2. **스마트 메모 허브** — [[Sense Makers]], [[Square]], My Note 등 모든 페이지에서 생성한 메모를 통합 관리

핵심 가치: 흩어진 감각 조각(메모)들을 모아서 연결하고, 더 깊은 기록(노트)으로 발전시키는 **감각의 교차점**.

---

## 3-1. 화면 레이아웃

2컬럼 레이아웃. 공통 [[헤더]]는 상단에 고정.

```
┌──────────────────────────────────────────────┐
│                  헤더 (10%)                    │
├────────────────────────┬─────────────────────┤
│                        │                     │
│    좌측 75%            │    우측 25%          │
│    노트 에디터          │    사이드바           │
│    (무지 / 양식지)       │    (내 노트 / 내 메모) │
│                        │                     │
│                        │                     │
└────────────────────────┴─────────────────────┘
```

---

## 3-2. 좌측 영역 (75%) — 노트 에디터

### 3-2-1. 에디터 툴바

상단 고정 (52px 높이). 3분할 구성.

```
┌─────────────────────────────────────────────┐
│ [+ 새 노트]     [무지 | 양식지]     [저장]   │
│  (좌측)           (중앙)            (우측)    │
└─────────────────────────────────────────────┘
```

| 위치 | 요소 | 설명 |
|------|------|------|
| 좌측 | 새 노트 버튼 | + 아이콘 + "새 노트" 텍스트, 클릭 시 에디터 초기화 |
| 중앙 | 타입 스위치 토글 | "무지" / "양식지" 2택 슬라이드 토글 |
| 우측 | 저장 버튼 | 저장 아이콘 + "저장" 텍스트, 저장 후 1.2초간 "저장됨 ✓" 표시 |

#### 타입 스위치 토글 동작

- 알약 형태의 토글 (pill-shaped toggle)
- 활성 쪽에 `--color-accent` 배경의 인디케이터가 슬라이드 이동
- 전환 시 해당 노트 타입의 에디터 화면으로 교체

#### 단축키

| 단축키 | 동작 |
|--------|------|
| `Ctrl/Cmd + S` | 노트 저장 |

---

### 3-2-2. 무지 노트 (Blank)

자유 형식 에디터. 제한 없이 생각을 흘려보내듯 기록.

```
┌─────────────────────────────────────┐
│ [제목을 입력하세요]                    │  ← input (maxlength: 100)
│                                     │
│ 자유롭게 생각을 기록하세요...           │  ← contenteditable div
│                                     │
│                                     │
│                                     │
│                                     │
└─────────────────────────────────────┘
```

| 요소 | 태그 | 스타일 |
|------|------|--------|
| 제목 | `<input type="text">` | Playfair Display, 3xl, bold |
| 본문 | `<div contenteditable>` | Pretendard, base, line-height 1.85 |

#### 텍스트 인터랙션

- 본문 텍스트 **드래그 가능** → 우클릭 → "Memo" 컨텍스트 메뉴
- 메모 생성 시 **옅은 노란색 하이라이트** 영구 적용 (Sense Makers와 동일 방식)
- 메모 선택 시 **진한 노란색**으로 변경 + 해당 위치로 자동 스크롤
- 상세 동작: [[메모 시스템]] 참고

### 3-2-3. 양식지 노트 (Template)

가이드 질문으로 감각을 구조화하여 기록.

```
┌─────────────────────────────────────┐
│ [제목을 입력하세요]                    │
│                                     │
│ Q1  오늘 가장 인상 깊었던 것은?        │
│ ┌─────────────────────────────────┐ │
│ │ (답변 textarea)                  │ │
│ └─────────────────────────────────┘ │
│                                     │
│ Q2  그것이 당신에게 어떤 의미를?       │
│ ┌─────────────────────────────────┐ │
│ │ (답변 textarea)                  │ │
│ └─────────────────────────────────┘ │
│                                     │
│ Q3  떠오르는 감각이나 감정은?          │
│ Q4  새롭게 알게 된 것은?              │
│ Q5  내일 해보고 싶은 것이 있다면?      │
└─────────────────────────────────────┘
```

| 요소 | 설명 |
|------|------|
| 질문 번호 | `Q1`~`Q5` 뱃지 (accent 배경, xs 크기) |
| 질문 텍스트 | base 크기, bold, primary 색상 |
| 답변 영역 | `<textarea>` surface 배경, border, 최소 80px 높이, 세로 리사이즈 가능 |

#### 5가지 가이드 질문

| # | 질문 |
|---|------|
| Q1 | 오늘 가장 인상 깊었던 것은 무엇인가요? |
| Q2 | 그것이 당신에게 어떤 의미를 주었나요? |
| Q3 | 떠오르는 감각이나 감정을 자유롭게 표현해보세요. |
| Q4 | 이 경험을 통해 새롭게 알게 된 것은? |
| Q5 | 내일 해보고 싶은 것이 있다면? |

> [!note] 고도화 예정
> - 양식지 템플릿 커스터마이징 (사용자 정의 질문)
> - 양식지 종류 추가 (독서 노트, 영화 감상, 음악 노트 등)

### 3-2-4. 프리뷰 패널

사이드바에서 **메모를 클릭**하면 에디터 영역이 **프리뷰 모드**로 전환되어 메모의 원본 출처 컨텐츠를 표시한다.

```
┌─────────────────────────────────────┐
│ [Sense Makers] 아티클 제목     [✕]  │  ← 프리뷰 헤더
├─────────────────────────────────────┤
│                                     │
│ 아티클 본문 텍스트...                 │
│ ████████████████████ ← 하이라이트    │
│ 본문 계속...                         │
│                                     │
└─────────────────────────────────────┘
```

| 요소 | 설명 |
|------|------|
| 소스 뱃지 | 메모 출처 표시 (My Note / Sense Makers / Square), 색상으로 구분 |
| 타이틀 | 원본 노트/아티클 제목 |
| 닫기 버튼 | 프리뷰 종료 → 에디터 모드로 복귀 |
| 본문 | 원본 컨텐츠, 발췌 텍스트 하이라이트 + 자동 스크롤 |

#### 소스별 프리뷰 동작

| 소스 | 뱃지 색상 | 프리뷰 내용 |
|------|----------|------------|
| My Note | accent 금색 | 원본 노트 내용 + 발췌 텍스트 하이라이트 |
| Sense Makers | 하늘색 (#7db8d4) | 아티클 본문 + 발췌 텍스트 하이라이트 |
| Square | 초록색 (#8fc06a) | 원본 메모 카드 (작성자 정보 포함) |

#### 프리뷰 상태 전환 규칙

```
에디터 모드 (기본)
    ↓  메모 클릭
프리뷰 모드 (툴바 숨김, 에디터 숨김, 프리뷰 표시)
    ↓  ✕ 버튼 또는 메모 선택 해제
에디터 모드로 복귀
```

## 3-3. 우측 영역 (25%) — 사이드바

### 3-3-1. 사이드바 탭

2개 탭으로 구성. 탭 전환 시 해당 패널만 표시.

```
┌────────────┬────────────┐
│  📄 내 노트  │  ✏ 내 메모  │
└────────────┴────────────┘
```

| 탭 | 아이콘 | 설명 |
|-----|-------|------|
| 내 노트 | 문서 아이콘 | 에디터에서 저장한 노트 목록 |
| 내 메모 | 편집 아이콘 | 모든 페이지에서 만든 메모 통합 관리 (스마트 메모 허브) |

- 활성 탭: `--color-accent` 색상 + 하단 보더
- 비활성 탭: `--color-text-muted` 색상

---

### 3-3-2. 내 노트 패널

에디터에서 저장한 노트들의 목록.

#### 노트 카드 구조

```
┌─────────────────────────────┐
│ [무지/양식지]           [✕]  │  ← 타입 뱃지 + 삭제 버튼
│ 제목                        │  ← 한줄 말줄임
│ 미리보기 텍스트...            │  ← 2줄 말줄임
│ 2026.02.26 14:30            │  ← 수정 일시
└─────────────────────────────┘
```

| 요소 | 설명 |
|------|------|
| 타입 뱃지 | "무지" (회색 배경) / "양식지" (accent 배경) |
| 삭제 버튼 | 호버 시에만 표시, 빨간색 호버 효과 |
| 제목 | 600 weight, 한줄 `text-overflow: ellipsis` |
| 미리보기 | 본문 첫 부분 or 답변들 · 로 연결, 2줄 `-webkit-line-clamp` |
| 수정 일시 | `yyyy.mm.dd HH:MM` 형식 |

#### 인터랙션

| 동작 | 결과 |
|------|------|
| 카드 클릭 | 해당 노트를 에디터에 로드 |
| 삭제(✕) 클릭 | 노트 삭제 (편집 중인 노트면 에디터 초기화) |
| 활성 노트 | accent 보더 + 엷은 accent 배경 |

#### 빈 상태

> 아직 작성한 노트가 없습니다.
> 왼쪽에서 첫 번째 노트를 작성해보세요.

### 3-3-3. 내 메모 패널 — 스마트 메모 허브

**모든 페이지에서 만든 메모를 한곳에서 관리**하는 통합 허브.

#### 패널 구조 (위 → 아래)

```
┌─────────────────────────────┐
│ 🔍  [메모 검색...]      [✕]  │  ← 검색바
├─────────────────────────────┤
│ [스트림] [타임라인] [소스별]   │  ← 뷰 모드 토글
├─────────────────────────────┤
│ [전체 ▾] [최신순 ▾] [🏷 ▾]   │  ← 컨트롤: 소스필터, 정렬, 태그필터
├─────────────────────────────┤
│ ★ 고정됨 (2)           ▾    │  ← 고정 섹션 (접기/펼치기)
│ ┌─ 고정 메모1 ─┐            │
│ └──────────────┘            │
├─────────────────────────────┤
│  ── 2/26 ──                 │  ← 날짜 구분선 (스트림 뷰)
│ ┌─ 메모 블럭 ──┐            │
│ └──────────────┘            │
│ ┌─ 메모 블럭 ──┐            │
│ └──────────────┘            │
│         [ + ]               │  ← 메모 추가 버튼
│                             │
│ (메모가 없으면 빈 상태 안내)   │
└─────────────────────────────┘
```

## 3-4. 메모 목록 형태 — 뷰 모드

3가지 뷰 모드 제공. 탭 형태 버튼으로 전환.

### 3-4-1. 스트림 뷰 (기본)

시간순 단일 리스트. 날짜가 바뀔 때마다 **날짜 구분선** 삽입.

```
── 2/26 ──
┌─ 메모 A ─┐
└──────────┘
┌─ 메모 B ─┐
└──────────┘
── 2/25 ──
┌─ 메모 C ─┐
└──────────┘
```

| 요소 | 설명 |
|------|------|
| 날짜 구분선 | 좌우 가로선 + 중앙 날짜 라벨 (월/일 형식) |
| 정렬 | 현재 정렬 설정(최신순/오래된순/수정순)에 따름 |

---

### 3-4-2. 타임라인 뷰

**시간 기반 그룹**으로 메모를 묶어 표시.

```
▾ 오늘 (3)
  ┌─ 메모 A ─┐
  ┌─ 메모 B ─┐
  ┌─ 메모 C ─┐

▾ 이번 주 (2)
  ┌─ 메모 D ─┐
  ┌─ 메모 E ─┐

▸ 이번 달 (5)  ← 접힘

▸ 이전 (12)
```

| 그룹 | 기준 |
|------|------|
| 오늘 | 오늘 00:00 이후 |
| 이번 주 | 이번 주 시작(일요일) 이후 |
| 이번 달 | 이번 달 1일 이후 |
| 이전 | 나머지 전체 |

#### 그룹 헤더

| 요소 | 설명 |
|------|------|
| 그룹 제목 | 10px, uppercase, 600 weight |
| 카운트 뱃지 | 9px, 라운드 배경 |
| 접기 화살표 | 클릭 시 그룹 내 메모 접기/펼치기 토글 |

---

### 3-4-3. 소스별 뷰

**메모 출처 페이지**별로 그룹화.

```
▾ My Note (5)
  ┌─ 메모 A ─┐
  ┌─ 메모 B ─┐

▾ Sense Makers (3)
  ┌─ 메모 C ─┐
  ┌─ 메모 D ─┐

▾ Square (2)
  ┌─ 메모 E ─┐
```

- 그룹 헤더 UI는 타임라인 뷰와 동일
- 그룹 접기/펼치기 지원

## 3-5. 연동 방식 — 크로스 페이지 메모 통합

### 3-5-1. 공유 저장소 구조

모든 페이지의 메모는 **하나의 공유 저장소**(`sensepoint_all_memos`)에 통합 저장된다.

```
localStorage
├── sensepoint_notes          ← My Note 전용 (노트 데이터)
└── sensepoint_all_memos      ← 전 페이지 공유 (메모 데이터)
    ├── source: 'my-note'     ← My Note에서 생성한 메모
    ├── source: 'sense-makers' ← Sense Makers에서 생성한 메모
    └── source: 'square'      ← Square에서 생성한 메모
```

### 3-5-2. 메모 데이터 구조

```javascript
{
  id: string,               // 고유 ID (예: 'mn_1_1708900000000')
  type: 'empty' | 'excerpt' | 'reply',
  source: 'my-note' | 'sense-makers' | 'square',
  
  // 메모 내용
  note: string,             // 사용자가 작성한 메모 텍스트
  sourceText: string,       // 발췌(인용)한 원본 텍스트
  
  // 출처 컨텍스트
  noteId: string,           // (my-note) 연결된 노트 ID
  noteTitle: string,        // (my-note) 노트 제목
  noteContent: string,      // (my-note) 노트 전문 (프리뷰용)
  articleTitle: string,     // (sense-makers) 아티클 제목
  articleContent: string,   // (sense-makers) 아티클 전문
  articleHtml: string,      // (sense-makers) 아티클 HTML
  originalAuthor: string,   // (square) 원본 메모 작성자
  
  // 하이라이트 위치
  range: {                  // contenteditable 내 범위 (null이면 하이라이트 없음)
    startContainer: number[],
    startOffset: number,
    endContainer: number[],
    endOffset: number,
    text: string
  },
  
  // 메타
  createdAt: number,        // 생성 시각 (timestamp)
  updatedAt: number,        // 수정 시각 (timestamp)
  visibility: 'private' | 'friends' | 'public',
  pinned: boolean,          // 고정 여부
  colorTag: string | null,  // 컬러 태그 ('coral' | 'blue' | 'green' | 'purple' | 'orange' | null)
  
  // 답글
  replies: [
    {
      id: string,
      author: string,
      avatarInitial: string,
      note: string,
      createdAt: number
    }
  ]
}
```

### 3-5-3. 페이지 간 동기화 흐름

```
Sense Makers 에서 메모 생성
    ↓  localStorage('sensepoint_all_memos')에 저장
    ↓  source: 'sense-makers' 태깅
    
My Note 페이지 진입
    ↓  sensepoint_all_memos 전체 로드
    ↓  모든 source의 메모를 통합 표시
    
My Note 에서 메모 수정/삭제
    ↓  sensepoint_all_memos 업데이트
    ↓  원본 페이지에서도 반영 (storage 이벤트 감지)
```

#### 실시간 동기화

- `window.addEventListener('storage')` — 다른 탭에서 메모 변경 시 자동 반영
- 같은 탭 내에서는 `syncSingleMemoToShared()` 함수로 즉시 동기화

### 3-5-4. 소스별 식별

| 소스 | 뱃지 표시 | 프리뷰 제공 데이터 |
|------|----------|-------------------|
| My Note | (뱃지 없음 — 기본) | noteTitle, noteContent |
| Sense Makers | `Sense Makers` 뱃지 (하늘색) | articleTitle, articleContent/articleHtml |
| Square | `Square` 뱃지 (초록색) | originalAuthor, sourceText |

> [!important] My Note 소스 메모에는 뱃지를 표시하지 않는다.
> 다른 페이지 출처의 메모에만 소스 뱃지를 표시하여 시각적으로 구분.

## 3-6. 검색 기능

### 3-6-1. 검색바 UI

접이식 검색바. 돋보기 아이콘 클릭으로 펼치기/접기.

```
접힌 상태:  [🔍]
펼친 상태:  [🔍] [메모 검색...              ] [✕]
```

| 상태 | 동작 |
|------|------|
| 접힌 상태 | 돋보기 아이콘만 표시 |
| 펼친 상태 | 입력 필드 + 초기화(✕) 버튼 표시 |

### 3-6-2. 검색 동작

| 속성 | 값 |
|------|-----|
| 검색 대상 | 메모 텍스트 (`note`), 발췌 텍스트 (`sourceText`), 노트/아티클 제목 |
| 디바운스 | 200ms (입력 완료 후 검색 실행) |
| 검색어 하이라이트 | 메모 블럭 내 일치 텍스트에 accent 배경 하이라이트 |
| 초기화 | ✕ 버튼 클릭 or ESC 키 → 검색어 삭제 + 전체 목록 복귀 |

### 3-6-3. 검색 범위

```
검색 쿼리 입력
    ↓
현재 적용된 소스 필터 + 태그 필터 범위 내에서 텍스트 매칭
    ↓
결과 메모만 표시 (현재 뷰 모드 유지)
```

검색은 **다른 필터와 조합**되어 작동한다:
- 소스 필터 "Sense Makers" + 검색어 "감각" → Sense Makers 소스 메모 중 "감각"이 포함된 것만 표시
- 태그 필터 "Coral" + 검색어 "영감" → Coral 태그 메모 중 "영감"이 포함된 것만 표시

## 3-7. 분류 기능

### 3-7-1. 소스 필터

메모 출처 페이지별 필터링.

| 옵션 | 표시 대상 | 카운트 |
|------|----------|--------|
| 전체 (기본) | 모든 메모 | 전체 메모 수 |
| My Note | `source === 'my-note'` | 해당 소스 메모 수 |
| Sense Makers | `source === 'sense-makers'` | 해당 소스 메모 수 |
| Square | `source === 'square'` | 해당 소스 메모 수 |

- 드롭다운 형태, 각 옵션 우측에 **카운트 숫자** 표시
- 활성 필터는 accent 색상으로 강조

### 3-7-2. 정렬

| 옵션 | 정렬 기준 |
|------|----------|
| 최신순 (기본) | `createdAt` 내림차순 |
| 오래된순 | `createdAt` 오름차순 |
| 최근 수정순 | `updatedAt` 내림차순 |

### 3-7-3. 컬러 태그 필터

메모에 지정된 색상 태그로 필터링.

| 옵션 | 색상 | CSS 변수 |
|------|------|---------|
| 전체 (기본) | — | — |
| Coral | 🔴 코랄 | `--color-tag-coral` |
| Blue | 🔵 파랑 | `--color-tag-blue` |
| Green | 🟢 초록 | `--color-tag-green` |
| Purple | 🟣 보라 | `--color-tag-purple` |
| Orange | 🟠 주황 | `--color-tag-orange` |
| 태그 없음 | — | `colorTag === null` |

- 드롭다운에 각 태그 색상 점(dot) 표시
- 태그 필터 활성 시 버튼에 accent 테두리로 필터 적용 상태 표시

### 3-7-4. 고정(Pin) 시스템

중요한 메모를 상단에 고정.

| 동작 | 방법 |
|------|------|
| 고정 | 메모 블럭 우클릭 → "고정" |
| 고정 해제 | 메모 블럭 우클릭 → "고정 해제" |

#### 고정 섹션 UI

```
★ 고정됨 (2)                      ▾
┌─ 고정 메모 1 (★ 아이콘 표시) ─┐
└──────────────────────────────┘
┌─ 고정 메모 2 ─┐
└──────────────┘
─────────────────────────────────
(일반 메모 목록)
```

- 고정된 메모가 1개 이상일 때만 고정 섹션 표시
- 섹션 헤더 클릭으로 접기/펼치기
- 고정 메모에는 ★ 아이콘 표시 (접힘 코너 우측)
- 고정 메모는 필터/정렬과 독립적으로 항상 상단 유지

### 3-7-5. 컬러 태그 지정

| 동작 | 방법 |
|------|------|
| 태그 지정 | 메모 블럭 우클릭 → "태그" → 색상 점 클릭 |
| 태그 제거 | 메모 블럭 우클릭 → "태그" → ✕ 점 클릭 |

- 메모 블럭 우측 상단에 해당 색상 점(6px 원) 표시
- 한 메모에 하나의 컬러 태그만 지정 가능

## 3-8. 메모 블럭 상세

### 3-8-1. 메모 블럭 구조

Sense Makers의 메모 블럭과 동일한 **포스트잇 스타일**. [[메모 시스템]] 참고.

```
┌──┬──────────────────────────────┐
│▚ │★  (deselect zone)        🔴 │  ← 접힘 코너 + 고정 아이콘 + 컬러 태그 점
├──┴──────────────────────────────┤
│ [Square] 작성자님의 메모 · 제목   │  ← 소스 뱃지 + 컨텍스트 (Square만)
│──────────────────────────────── │
│ 메모 내용 입력 영역               │  ← seamless textarea
│                                 │
│ ── 답글들 ──                     │  ← replies (선택 시에만 표시)
│ 메모에 메모하기                    │  ← reply action (선택 시에만 표시)
│                                 │
│ [🔒 나만보기]             26.02  │  ← 공개 범위 + 날짜
└─────────────────────────────────┘
```

### 3-8-2. 메모 블럭 크기

| 상태 | 크기 | 동작 |
|------|------|------|
| default | `--memo-block-height` (5개 블럭 수용) | 답글/메모에메모 숨김, 내용 잘림 |
| 선택 (클릭) | max-height 해제 | 답글 표시, 메모에메모 표시, 전체 내용 표시 |

### 3-8-3. 공개 범위 설정

메모 블럭 하단 좌측의 공개 범위 버튼.

| 옵션 | 아이콘 | 설명 | 상태 |
|------|--------|------|------|
| 나만보기 | 🔒 자물쇠 | 본인만 열람 가능 | 기본값 |
| 친구랑 보기 | 👥 사람들 | 친구 관계인 사용자도 열람 | 준비중 (비활성) |
| 다같이 보기 | 🌐 지구본 | Square에 공개 | 활성 |

- 클릭 시 상단 방향 드롭다운 펼침
- "친구랑 보기"는 `준비중` 뱃지 표시 + 클릭 불가
- 변경 즉시 공유 저장소에 동기화

### 3-8-4. 메모에 메모하기 (답글)

메모 블럭 **선택 상태**에서만 표시.

```
[💬 메모에 메모하기]          ← 버튼 (클릭 시 폼 전환)

┌─────────────────────────┐
│ 이 메모에 대한 생각을...   │  ← textarea
├─────────────────────────┤
│              [취소] [등록] │  ← 액션 버튼
└─────────────────────────┘
```

| 동작 | 설명 |
|------|------|
| Enter | 등록 |
| Shift+Enter | 줄바꿈 |
| Escape | 취소 |

#### 답글 표시

```
├─ 나 · 26.02
│  이 부분이 인상적이었어요
├─ 나 · 26.02
│  다시 읽어봐야겠다
```

- 왼쪽 세로선 (accent 20% 색상)
- 아바타(16px 원) + 작성자 + 시간
- 메모 텍스트

### 3-8-5. 컨텍스트 메뉴 (우클릭)

메모 블럭 우클릭 시 표시.

```
┌─────────────────────────┐
│ ★ 고정 / 고정 해제       │
│ 🏷 태그  [🔴🔵🟢🟣🟠✕]  │
│ ─────────────────────── │
│ 🗑 삭제                  │
└─────────────────────────┘
```

## 3-9. 메모 생성 방식

### 3-9-1. 텍스트 드래그 → 우클릭 → "Memo"

무지 노트 본문(contenteditable) 또는 양식지 답변(textarea)에서 텍스트 드래그 후 우클릭.

| 단계 | 동작 |
|------|------|
| 1 | 에디터 본문에서 텍스트 드래그 |
| 2 | 우클릭 → "Memo" 컨텍스트 메뉴 표시 |
| 3 | "Memo" 클릭 |
| 4 | 메모 블럭 생성 (type: 'excerpt', 발췌 텍스트 포함) |
| 5 | 무지 노트인 경우 본문에 하이라이트 적용 |
| 6 | 사이드바 "내 메모" 탭 자동 전환 |
| 7 | 새 메모 자동 선택 |

### 3-9-2. "+" 버튼 클릭

사이드바 하단의 추가 버튼.

| 단계 | 동작 |
|------|------|
| 1 | "+" 버튼 클릭 |
| 2 | 빈 메모 블럭 생성 (type: 'empty') |
| 3 | 새 메모 자동 선택 + textarea 포커스 |

#### "+" 버튼 스타일

| 속성 | 값 |
|------|-----|
| 형태 | 점선 라운드 박스 |
| 텍스트 | `+` (2xl 크기, 중앙) |
| 기본 | 투명 배경 + muted 테두리 |
| 호버 | 음영 배경 + 더 진한 테두리 |

## 3-10. 노트 CRUD

### 3-10-1. 생성

| 트리거 | 결과 |
|--------|------|
| "새 노트" 버튼 클릭 | 에디터 초기화 + 무지 모드 + 제목 포커스 |

### 3-10-2. 저장

| 조건 | 동작 |
|------|------|
| 제목 또는 내용 있음 | 저장 실행 |
| 모두 비어있음 | 저장 무시 |
| 기존 노트 수정 | `updatedAt` 갱신 |
| 새 노트 | ID 생성 + `notes` 배열 맨 앞에 추가 |

저장 후:
1. localStorage에 반영
2. 사이드바 노트 목록 갱신
3. 저장 버튼 "저장됨 ✓" 피드백 (1.2초)

### 3-10-3. 로드

사이드바 노트 카드 클릭 시:
1. `currentNoteId` 업데이트
2. 노트 타입에 맞는 에디터 전환
3. 에디터에 데이터 채우기
4. 사이드바 활성 카드 하이라이트

### 3-10-4. 삭제

사이드바 노트 카드 삭제(✕) 클릭 시:
1. `notes` 배열에서 제거
2. 현재 편집 중인 노트면 에디터 초기화
3. localStorage 반영
4. 사이드바 목록 갱신
