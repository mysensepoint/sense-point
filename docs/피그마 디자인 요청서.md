## 5. 페이지별 상세 스펙

### 5-1. 랜딩 페이지 (index.html)

**구조:**
```
┌──────────────────────────────────────────────┐
│  헤더 (높이 10vh)                              │
│  [SP 로고]                      [≡ 메뉴 버튼]  │
├──────────────────────────────────────────────┤
│                                              │
│  메인 영역 (높이 90vh) — 배너 캐러셀              │
│                                              │
│   태그: "Discover Your Senses"               │
│   제목: "감각을 깨우는                          │
│         새로운 경험"                           │
│   설명: "SensePoint는..."                     │
│   [CTA 버튼: Explore Now]                     │
│                                              │
│                          [← 01 / 03 →]       │
│                          (우측 하단 네비게이션)   │
└──────────────────────────────────────────────┘
```

**디자인 포인트:**
- 배너는 3장의 풀스크린 캐러셀 (좌우 드래그 + 화살표 이동)
- 배경은 그라디언트 (이미지 사용도 가능)
- 물결(Water Ripple) 효과가 위에 겹침 (Canvas) — 디자인에서는 무시해도 됨
- 텍스트는 좌측 정렬 (모바일에서는 중앙 정렬)
- 네비게이션 화살표 + 카운터는 우측 하단 (모바일에서는 하단 중앙)

### 5-2. Sense Makers (sense-makers.html)

**구조:**
```
┌──────────────────────────────────────────────┐
│  헤더 (10vh)                                  │
├────────────────────────┬─────────────────────┤
│                        │                     │
│  좌측 75%              │  우측 25%             │
│  아티클 본문            │  메모 사이드바          │
│                        │                     │
│  [카테고리 태그]         │  ┌─────────────┐    │
│  제목 (큰 글씨)         │  │ ▚ 메모 블럭 #1 │    │
│  작성자 · 날짜          │  │ "인용 텍스트" │    │
│                        │  │ 메모 내용     │    │
│  본문 텍스트 ...        │  │        26.02 │    │
│  (드래그하여 메모 생성)   │  └─────────────┘    │
│                        │  ┌─────────────┐    │
│                        │  │ 메모 블럭 #2  │    │
│                        │  └─────────────┘    │
│                        │      [ + ]          │
└────────────────────────┴─────────────────────┘
```

**디자인 포인트:**
- 2컬럼 레이아웃 (PC: 75/25, 태블릿: 70/30, 모바일: 단일 컬럼)
- 메모 블럭은 **포스트잇 느낌** — 따뜻한 톤, 그림자, 왼쪽 상단 접힌 코너
- 메모가 없을 때: "Memo" 타이틀(24pt) + 설명(14pt) + "+" 버튼
- 메모 생성 후: 타이틀/설명 숨김, 메모 블럭만 표시
- 하이라이트 색상: 옅은 노란색(기본) / 진한 노란색(선택 중)
- 메모 블럭 기본 크기: 사이드바에 5개가 들어가는 크기
- 메모 블럭 선택 시: 10줄 텍스트가 보이는 크기로 확장

## 6. 공통 컴포넌트

### 6-1. 헤더 ✅ 피그마 적용 완료

```
┌──────────────────────────────────────────────┐
│  [로고 SVG 146x24]              [≡ 24x24]     │
│  (img/logo-horizontal.svg)      (햄버거/X)    │

```
- 높이: 56px 고정 (`padding: 16px 36px`)

### 6-3. 메모 블럭 (포스트잇)

```
┌──┬──────────────────┐
│▚ │                  │  ← 왼쪽 상단 접힌 코너 (14px)
├──┴──────────────────┤
│ "인용된 텍스트..."     │  ← 발췌 (있을 경우)
│ 메모 내용             │  ← 입력 영역 (경계선 없음)
```

- **2가지 상태** 디자인 필요:
  - Default: 접혀있는 상태 (작은 크기)
  - Selected: 펼쳐진 상태 (10줄 텍스트 보이는 크기)
- 왼쪽 상단 접힌 코너: 드래그 핸들 역할
- 상단 20px 영역: 선택 해제 존 (호버 시 음영 구분)

## 7. 피그마 작업 시 지켜주면 좋은 것들 ⭐

### 7-1. Auto Layout 사용
- 모든 프레임에 **Auto Layout** 적용해줘
- 코드가 Flexbox 기반이라 Auto Layout이 1:1로 대응됨
- 고정값(px)보다 **비율(%)** 기반으로 만들어줘

### 7-2. 프레임/레이어 이름 규칙
피그마에서 이 이름 규칙을 따르면 코드 적용이 쉬워:

| 피그마 프레임 이름 | 대응하는 코드 |
|-----------------|-------------|
| `Header` | `.header` |
| `Header/Logo` | `.header__logo` |
| `Header/MenuBtn` | `.header__menu-btn` |
| `MenuPanel` | `.menu-panel` |
| `Landing/Banner` | `.landing__banner` |
| `Landing/Banner/Slide` | `.landing__banner-slide` |
| `SenseMakers/Article` | `.sense-makers__article` |
| `SenseMakers/Sidebar` | `.sense-makers__sidebar` |
| `MemoBlock` | `.memo-block` |
| `MemoBlock/Fold` | `.memo-block__fold` |
| `MemoBlock/Content` | `.memo-block__content` |
| `Square/Feed` | `.square__feed` |
| `Square/AuthorSidebar` | `.square__author-sidebar` |

> 꼭 이 이름이 아니어도 되지만, **일관된 네이밍**을 쓰면 OK.

### 7-3. 색상/텍스트를 피그마 스타일로 등록
- 색상 팔레트를 **Color Styles**로 등록
- 텍스트 크기/폰트를 **Text Styles**로 등록
- 나중에 변경사항을 한눈에 확인 가능

### 7-4. 컴포넌트화
다음 요소들은 피그마 **Component**로 만들어줘 (재사용):
- 헤더
- 메뉴 오버레이
- 메모 블럭 (Default / Selected 상태)
- 메모 추가 버튼 (+)
- 필터 탭 (모두 보기 / 친구 메모)
- 메모 카드 (Square용)

### 7-5. 인터랙션 상태 표시
가능하면 아래 상태별 디자인도 포함:
- **Default** / **Hover** / **Active** / **Disabled**
- 메모 블럭: Default / Selected / 드래그 중
- 버튼: Default / Hover / Active

## 8. 디자인 완료 후 전달 방법

완성된 디자인을 코드에 적용할 때 아래 정보가 필요해:

### 방법 1: 스크린샷 + 수치 (가장 간편)
1. 각 페이지/컴포넌트 **스크린샷** 캡처
2. 변경된 색상값, 폰트 크기, 간격 등 **주요 수치** 메모
3. 스크린샷과 수치를 함께 전달

### 방법 2: 피그마 Dev Mode (가장 정확)
1. 피그마에서 **Dev Mode** 활성화 (`</>` 버튼)
2. 각 요소의 **CSS 값** 복사 가능
3. 공유 링크 전달하면 코드에서 직접 참조 가능

### 방법 3: 피그마 공유 링크
1. 피그마 파일 **View 권한 공유**
2. 링크만 전달하면 Cursor AI가 참고하여 적용
